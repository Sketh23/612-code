\subsection{Detailed Description}
A \hyperlink{classCommandGroup}{CommandGroup} is a list of commands which are executed in sequence.

Commands in a \hyperlink{classCommandGroup}{CommandGroup} are added using the \hyperlink{}{AddSequential(...)} method and are called sequentially. \hyperlink{classCommandGroup}{CommandGroups} are themselves \hyperlink{classCommand}{Commands} and can be given to other \hyperlink{classCommandGroup}{CommandGroups}.

\hyperlink{classCommandGroup}{CommandGroups} will carry all of the requirements of their \hyperlink{classCommand}{subcommands}. Additional requirements can be specified by calling \hyperlink{}{Requires(...)} normally in the constructor.

CommandGroups can also execute commands in parallel, simply by adding them using \hyperlink{}{AddParallel(...)}.

Adds a new child \hyperlink{classCommand}{Command} to the group with the given timeout. The \hyperlink{classCommand}{Command} will be started after all the previously added \hyperlink{classCommand}{Commands}.

Once the \hyperlink{classCommand}{Command} is started, it will run until it finishes, is interrupted, or the time expires, whichever is sooner. Note that the given \hyperlink{classCommand}{Command} will have no knowledge that it is on a timer.

Instead of waiting for the child to finish, a \hyperlink{classCommandGroup}{CommandGroup} will have it run at the same time as the subsequent \hyperlink{classCommand}{Commands}. The child will run until either it finishes, the timeout expires, a new child with conflicting requirements is started, or the main sequence runs a \hyperlink{classCommand}{Command} with conflicting requirements. In the latter two cases, the child will be canceled even if it says it can't be interrupted.

Note that any requirements the given \hyperlink{classCommand}{Command} has will be added to the group. For this reason, a \hyperlink{classCommand}{Command's} requirements can not be changed after being added to a group.

It is recommended that this method be called in the constructor.
